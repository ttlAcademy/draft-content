Set up Java: Introduction

Welcome

Hi there, welcome to the Java set-up. This set-up serves as prerequisite for all the courses that make use of Java Courses. We are going to make sure you can start writing your application after this tutorial.

Before jump into the installation, it is better to understand Java Development Environment.

JRE

JRE stands for Java Runtime Environment. Without JRE we cannot run Java programs. It contains runtime libraries and Java application launcher that are necessary to run programs written in Java Language.

JDK

If you have only JRE, you can run Java programs but you can not develop application. For this purposes we need JDK. JDK stands for Java Development Kit.The JDK is a development environment for building applications, applets, and components using the Java programming language. JDK includes JRE plus the development tools (such as compiler and debugger). Because JRE is inside JDK, you should install JDK to both write and run Java programs.

Previous Set-up

If you think that in the past you’ve set-up Java but are not sure if your set-up is correct, follow the instruction according to your operation system.

We recommend that you have version 8 since it is very common and many of the project still it is in use. On the other hand Java SDK 11 is the latest stable version and has long term support (LTS) version at the moment we have written this tutorial. Although you can install multiple versions of JDK/JRE concurrently, it can be easily messy. Which version should you use depends on your environments and project requirements. For example if you explicitly use VS Code as an IDE you need to use Java SE 11 to completely setup and run your code. If you are using a test framework, many of them work well together with Java version 8. All in all the aim is here to show you how you can setup correctly your environment. If you have Java 8 or 11 installed, you can skip this tutorial.

On Windows

Open PowerShell and try executing the following statements to check it out.

javac -version

If you get a response with the version number and it looks somewhat like the following output, it means you have Java SDK installed.

javac 11.0.6

Look at version above, 11. This version is known as Java 11

If you get an error message like the following one, your setup is not correct and you should follow this course and move forward.

PS C:\>javac -version
The term 'javac' is not recognized as the name of a cmdlet, function, script file, or operable program.
Check the spelling of the name, or if a path was included, verify that the path is correct and try again.

Uninstall Old Version From Windows

If you have older version than version Java 8, we recommend uninstall this version from your PC before proceeding with this course unless you explicitly need older version for some reasons. To do this, open the start menu, type “programs” and click on “add or remove programs”

In the apps & features list, look for java and remove all versions of java other than version 8 or version 11.

To do this, select the java version and click on uninstall. Repeat this process until all older java installations are removed, once done, you can proceed to the next lesson.

On Ubuntu

We are explaining how you can do same steps in linux, based on Ubuntu distribution. These steps are applicable both Ubuntu version 18.04 and 20.04. Open your terminal and try executing the following statements to verify whether the JDK is installed or not.

javac --version

If you get a response with the version number and it looks somewhat like the following output, it means you have Java SDK installed.

javac 11.0.7

Look at version above, 11. This version is known as Java 11

If you get an warning message like the following one, you haven't setup java and you should follow this course. 

javac --version

Command 'javac' not found, but can be installed with:

sudo apt install openjdk-11-jdk-headless  # version 11.0.9+11-0ubuntu1~20.04, or
sudo apt install default-jdk              # version 2:1.11-72
sudo apt install ecj                      # version 3.16.0-1
sudo apt install openjdk-8-jdk-headless   # version 8u272-b10-0ubuntu1~20.04
sudo apt install openjdk-13-jdk-headless  # version 13.0.4+8-1~20.04
sudo apt install openjdk-14-jdk-headless  # version 14.0.2+12-1~20.04

If you have older version than version Java 8, we recommend you to set version 8 as an default version. We will be explaining following lessons how to manage your Java versions.
