What is Java?

Java is a class-based, object-oriented programming language which allows a developer to define a type of data with its set of assets and operation. It enable us to write code once and run anywhere (WORA). It means it is platform-independent. Java code can run on all platforms. But how this is possible, we will explain in the following lines.

IDE --> JDK --> JRE --> JVM

Java code is text file with .java extension nothing more nothing else. But the power of java is its structure. As we mentioned its platform-independent. Java program needs Java Runtime Environment (JRE) to be installed.

JRE

JRE stands for Java Runtime Environment. Without JRE we cannot run Java programs. It contains runtime libraries and Java application launcher that are necessary to run programs written in Java Language. The most important part of JRE is Java Virtual Machine (JVM).

JVM

JVM creates an environment in which Java programs interact with hardware no matter operation system is. This is the minimum requirement to run Java. If we want to develop a Java application then we need Java Development Kit(JDK).

JDK

If you have only JRE, you can run Java programs but you can not develop applications. For this purpose we need JDK. JDK is a development environment for building applications, applets, and components using the Java programming language. JDK includes JRE plus the development tools (such as compiler and debugger). The good thing is once you install JDK you don't have to install JRE again.

Once developer creates source code as .java file, an executable java program generated from the source code and this is called Java bytecode. Java bytecode file extension is .class. JDK is responsible to this generation.

IDE

Of course we need a text editor to write our source code (.java files.) We can even use notepad, but popular code editors come with tons of handy tools and extension to make programming easy. Since they understand the syntax of our code and offer different type of assistance such as debugging, code completion, code intelliSense, they are more than a text editor. Literally they are called Integrated Development Environment (IDE). Popular IDEs for Java include IntelliJ, Eclips, Netbeans and nowadays Visual Studio Code.

Our First Java Program, "Hello Java"

Now we can observe Java in action. For this purpose let us start writing our first Java program. It will be popular Hello, Java! codes. You may use your editor or IDE but simple notepad is enough for the first time. Open your editor and write down below codes and save as HelloJava.java

public class HelloJava {
    public static void main ( String [] args ) {
        System .out. println ( "Hello , Java!" );
    }
}

Executing this codes produces a simple single line printout: Hello, Java!Using this example we are going to learn some important features of Java programs and mention a bit more on how compilation works in Java.

The Source Code and Compilation

After saving your first Java files open your terminal and go to directory in which your the HelloJava.java is saved. To compile the source code HelloJava.java in a command line interface, type below command: 

javac HelloJava.java

If you don't get an ERROR, a file named HelloJava.class is created. This is the bytecode of the our program and it is called class file. If you compiled before and the same class file already exists under the same directory, then the file is overwritten after new successful compilation. Open your file explorer, go to your directory and make sure the class file was compiled. Now we can execute this bytecode write below command:

java HelloJava

This command execute our .class file and produces output: Hello, Java!

Let's come back to our source code and please focus on first line public class HelloJava. This code states that: (a) this is a class, (b) the class name is HelloJava, (c) this class is accessible from all other Java program components. The keyword public specifies that the class is accessible from other program units. These types of keyword called as access modifiers. There are three modifiers in Java: public, private, and protected. If you don't specify modifiers, the default modifier is taken from Java package. Every source file in Java has the file extension .java. In a single .java file, we can define more than one class. In such a case, at most one class may be public. if a source file contains a class with the public access modifier, the name of the class must match the file name. For example, 

public class Foo
{
...
}
class Bar
{
...
}

is acceptable as long as the file name is Foo.java. Yes, if we have public class in our java file, this file's name should match with public class name. Let us test this rule with our Hello Java program. If the class name is changed from HelloJava to just Hello while the file name remains the same, HelloJava.java:

public class Hello {
    public static void main ( String [] args ) {
        System .out. println ( "Hello , Java!" );
    }
}

if you compile the source code (javac HelloJava.java), compile the compilation produces the following error message:

$ javac HelloJava.java
HelloJava.java:1: error: class Hello is public, should be declared in a file named Hello.java
public class Hello {
       ^
1 error

The first statement of the error message state the cause of the error, which is that, to define a class by the name of Hello, the file name must be Hello.java. The next statement specify where is the error occurred using the “caret” symbol ^. It tells us the error is at the class declaration. The last line of the error message is the total number of errors found in the source code.
