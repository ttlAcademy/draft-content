Exercise 4.1: Pilot & deployment guidelines sample solution.

Learning Objective ALTA-E 4.1.1 (K3) Apply guidelines that support effective test tool pilot and deployment activities.
Version 1.1

  1. Identify a suitable pilot project. State why you think it is the best project to select as a pilot.

    Group A is probably the best candidate for a pilot project:

    • Applications don't need to be changed.
    • Defect tracker (Jira) is the same, so only small integration issues need to be resolved.
    • Development is different (SVN vs GitHub, Java vs PHP, Eclipse vs Visual Studio). These differences warrant the effort to have a full pilot.

    Group B is just formed, without developers, so all development work would rest on your shoulders. This may include setting up the defect tracker, etc.

    Group C wants a complete rewrite of your applications, so this would not be a pilot project, but the creation of a new TAS.

  2. In short: pilot will be successful when Group A will be able to test their application with minimal extra effort

    More detailed:

    • TAS is adapted for their processes and tools where needed.
    • The tester can easily write test cases in the TAS.
    • Tests run automated at least once per day with minimal effort.

  3. Sequence of projects to deploy your TAS.

    Group A > Group B > Group C

  4. Focus points and success factors for each subsequent deployment.

    Group A: see answer 2

    Group B:

    • Focus on the rewrite of the webservice, extend TAS to accommodate the added functionality.
    • Integration with defect tracker.
    • Make sure to keep up with the new developments.

    Group C:

    • Since Group C plans to rewrite the webservice and WebApp, the only focus may be MailingApp and Admin.
    • You should make sure you are involved when Group C starts to select a development environment, to make sure the TAS fits with minimal effort.
