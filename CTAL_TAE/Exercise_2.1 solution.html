Test Tool Requirements Weight Explanation
SUT interface: Browser Testing Must Core application is SPA and even on mobile devices web technologies (Ionic Framework) will run for the sake of HTML support for media and cross-platform support with optimal performance. So browser testing has to be in the main effort.
SUT Interface: Mobile Devices Must New application easily can feed mobile devices through API. Since many people are using mobile devices more than PC for their daily life, mobile application test should be in place.
SUT interface: API Testing Must API will be ain action both internal and external services including third party email services. So this is definitely one of the most important part of the new application.
SUT Database access Should All information will be stored in MySQL database. Test team deal with data through interfaces and mainly data test will be black box test. But to make sure data is consist across the critical components such as stock control, orders, refund etc. test team needs to compare raw data and should able to query to the served data by application. Without compromising access rights policy, proper data access should be in place for testing. For example access to mock snapshot.
Third Party: Integration with Jira Should Defect tracking and reporting are run by Jira. Test roadmap aligned with development roadmap should be integrated into Jira. So possible tools are required to have features for this integration.
SUT Architecture: Support Full JS Environment (Node / Express) Must New application is being built by VENOMY (Vue, Express; Node and MySQL) stack and both back and front side JS is used, support for SUT Architecture shouldn't be a problem for any modern test framework.
SUT size & complexity: Must New Chinook platform is mid-size enterprise web application. Possible tools should start simple and can be extend easily.
OS Support: Linux Must Development team mainly uses linux based platform for their development and developer uses MAC PC. So unix based support should be "MUST".
